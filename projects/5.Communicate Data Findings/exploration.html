def get_file(date):
    global local_path, bike_data_local
    # the file names are renamed after this date
    if date < '201905':
        file_name = zip_file.format(date, 'fordgobike')
    else:
        file_name = zip_file.format(date, 'baywheels')

    local_file = local_path + str(date) + 'local.csv.gz'
    df = pd.DataFrame([])

    if os.path.isfile(local_file):
        print('Fetch local month ' + date)
        df = pd.read_csv(local_file, low_memory=False, compression='gzip')
    else:
        print ("Downloading: " + zip_url.format(file_name))
        try:
            header = {'user-agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/79.0.3945.130 Safari/537.36',}
            remotezip = requests.get(zip_url.format(file_name), headers=header)
            root = zipfile.ZipFile(io.BytesIO(remotezip.content))
            df = pd.read_csv(root.open(file_name), low_memory=False, usecols=csv_cols)
            df.to_csv(local_file, index=False, compression='gzip')
        except: 
            exc_type, exc_value, exc_traceback = sys.exc_info()
            traceback.print_tb(exc_traceback, limit=1, file=sys.stdout)
            traceback.print_exception(exc_type, exc_value, exc_traceback,
                                  limit=2, file=sys.stdout)

    return df

if os.path.isfile(bike_data_local):
    print('Load from local {} months file'.format(months_count))
    dfall = pd.read_csv(bike_data_local, low_memory=False, compression='gzip')
else:

    df_list = []
    for i in range(0, months_count):
        crt_date = start_date + relativedelta(months=i)
        df = get_file(crt_date.strftime('%Y%m'))
        df_list.append(df)
        print(df.shape)
    print('Contatenate all months into one file')
    dfall = pd.concat(df_list)
    print('Save {} months file'.format(months_count))
    dfall.to_csv(bike_data_local, index=False, compression='gzip')

Assessing Data¶

During the assesment process we will check for two aspects of the gathered data: quality and tidiness.
The quality refers to the issues we have with the content of the data: inaccurate data, corrupted data or duplicated data. Tidiness is related to the structure of the data, Tidy data requires:

  • each variable forms a column.
  • each observation forms a row.
  • each type of observational unit forms a table.

Structure of the dataset¶

  • duration_sec - int64 - Duration of the ride in seconds
  • start_time - object - The start time of the ride
  • end_time - object - The end time of the ride
  • start_station_id - float64 - The start station id or Null if not started in a station
  • start_station_name - object - The start station name
  • start_station_latitude - float64 - The start station latitude
  • start_station_longitude - float64 - The start station longitude
  • end_station_id - float64 - The end station id or Null if not left the bike in a station
  • end_station_name - object - The end station name
  • end_station_latitude - float64 - The end station latitude
  • end_station_longitude - float64 - The end station longitude
  • bike_id - int64 - The bike id
  • user_type - object - A value determining if the user is a Subscriber or Customer
  • bike_share_for_all_trip - object - Yes|No user is in the "Bake Sharing for all" promotion

Features of interest¶

I will use in my analysis all of the features in the dataset. In order to see details about trips will have to use start_time, end_time, duration_sec and derived features hour, day, weekday). For answering questions about the bikes I will use bike_id, start_station_id. To find how many subscribers there are and if they use some offers I will use user_type and bike_share_for_all_trip. I will use start_station_id, start_station_name, end_station_id, end_station_name to compute stats about stations. In order to identify the start_city and end_city I will use start_station_latitude, start_station_longitude, end_station_latitude, end_station_longitude
